Visión general

Este documento describe el proyecto web-antispoofing y su SDK AntiSpoofing, enfocado en detección facial en el navegador, validaciones (centrado, luz, proporciones, orientación y mirada al frente), y validación final contra un servicio externo.

Qué hace
  • Lee un MediaStream (cámara) y analiza el video con face-api.js.
  • Aplica validaciones en tiempo real y guía al usuario con mensajes (advise, error).
  • Captura N fotogramas en base64 y los envía a urlApi.
  • Emite un resultado final (response) indicando si hay spoof o no.
Lo que NO hace
  • No administra permisos de cámara (eso lo hace tu app con getUserMedia).
  • No incluye el backend: el endpoint urlApi debe existir y cumplir el contrato.
  • La mayoría de validadores son internos (marcados como private en TypeScript).

Quickstart

1) Incluir el SDK (build UMD)

El servidor de este repo compila el TypeScript y expone un bundle UMD como global AntiSpoofing.

<script src="/static/sdk/web-antispoofing-sdk.js"></script>

2) HTML mínimo

<video id="video" autoplay playsinline muted></video>
<canvas id="canvas" style="display:none"></canvas>

3) Inicialización

(async function () {
  const video = document.getElementById("video");
  const canvas = document.getElementById("canvas");
  const stream = await navigator.mediaDevices.getUserMedia({ video: true, audio: false });

  const sdk = new AntiSpoofing({
    videoElement: video,
    canvasElement: canvas,
    stream,
    inputSize: 512,
    scoreThreshold: 0.5,
    modelsRoute: "/models",
    urlApi: "https://TU_BACKEND/validate",
    originSelfHeader: "TU_ORIGEN_O_TOKEN",
    actionListener: (action, data) => {
      switch (action) {
        case "advise":
        case "success":
          console.log("[UI]", String(data));
          break;
        case "error":
          console.error("[SDK ERROR]", String(data));
          break;
        case "framesCaptured":
          console.log("Frames capturados:", data); // string[]
          break;
        case "response":
          console.log("Resultado:", data); // ApiResponse
          break;
      }
    },
  });

  // Nota: el SDK arranca su flujo internamente al construirse (configureSDK()).
})();

Eventos (actionListener)

El SDK notifica estados y resultados a través de actionListener(action, data). El parámetro action es uno de: advise, error, framesCaptured, response, success.

actiondataCuándo ocurreEjemplos
advise string Guía al usuario cuando alguna validación falla (luz, distancia, orientación, múltiples rostros, etc.). "Se requiere más luz en el rostro", "Mira hacia al frente"
success string Estado informativo mientras se capturan los fotogramas. "Por favor no te muevas…"
error string Errores de carga o de flujo (por ejemplo, no hay rostro detectado). "Error al cargar el SDK: ..."
framesCaptured string[] Cuando se alcanzan los fotogramas requeridos (framesLength, por defecto 10) y se envían al backend. ["data:image/jpeg;base64,...", ...]
response ApiResponse Al finalizar la evaluación del backend. { success: true, isSpoof: false }

Nota importante sobre modelos
En el código actual, loadModels() carga TinyFaceDetector desde "/models" (ruta fija), mientras que modelsRoute se usa para loadFaceLandmarkModel(modelsRoute). Si tu despliegue usa una ruta distinta, considera unificar ambas rutas en el código.

Variables de entorno

Se lee PORT (por defecto 4000). Ajusta según tu despliegue.

Contrato del servicio (urlApi)

Al completar la captura, el SDK hace un POST a urlApi con JSON y header origin_self.

Request

{
  "imgs_base64": [
    "BASE64_SIN_PREFIJO_data:image",
    "..."
  ]
}

El SDK remueve el prefijo data:image/(png|jpg|jpeg);base64, antes de enviar.

Response esperada

{
  "cod_error": 0,
  "data": {
    "analisis": "Spoof"
  }
}

Interpretación

  • cod_error === 0 y analisis === "Spoof" → { success:true, isSpoof:true }
  • cod_error === 0 y analisis !== "Spoof" → { success:true, isSpoof:false }
  • cod_error !== 0 → { success:false, isSpoof:null }

Troubleshooting

  • No se detecta rostro: valida permisos de cámara, iluminación y que el video tenga autoplay + playsinline.
  • Error al cargar modelos: verifica que /models exista y no haya 404 en la pestaña Network.
  • CORS al llamar urlApi: habilita CORS en tu backend (o usa un proxy).
  • Mensajes constantes de distancia: ajusta porcentajes min/max del tamaño de rostro en el código si tu UI tiene otro encuadre.

Notas de seguridad y privacidad

  • Usa HTTPS siempre (las fotos viajan como base64).
  • No loguees frames completos en producción.
  • Define retención/borrrado en backend y consentimiento del usuario.
  • Si origin_self es un token, asume que es un secreto.
